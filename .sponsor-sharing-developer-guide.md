# Sponsor Sharing UI - Developer Guide

## Quick Start

### 1. Journal Entry Sharing

Users can share journal entries with their sponsor directly from the journal list:

```typescript
// Already integrated in JournalListScreenModern.tsx
import { ShareEntryModal } from '../components/ShareEntryModal';

// Usage:
<ShareEntryModal
  visible={shareModalVisible}
  entry={selectedEntry}
  onClose={() => setShareModalVisible(false)}
  onSuccess={() => refetch()}
/>
```

**User Flow**:
1. User taps share icon on journal entry
2. Modal opens with entry preview
3. User selects sponsor (if multiple, usually just one)
4. User taps "Share"
5. Encrypted payload generated
6. User taps "Send to Sponsor" → system share dialog opens
7. User shares via SMS/WhatsApp/etc.

### 2. Sponsor Dashboard

The main hub for sponsors to view their sponsees:

```typescript
import { SponsorScreenModern } from '../features/sponsor/screens/SponsorScreenModern';

// Automatically shows:
// - Sponsor/sponsee stats
// - Quick action cards
// - List of connections
// - Unread entry badges
```

**Features**:
- Stats row: sponsor count, sponsees count, unread count
- Quick actions: "Find Sponsor", "Be a Sponsor"
- Connection list with unread badges
- Empty state guidance
- Pending invites notice

### 3. Viewing Shared Entries

Sponsors can view entries shared by their sponsees:

```typescript
import { SharedEntriesScreenModern } from '../features/sponsor/screens/SharedEntriesScreenModern';

// Route params:
navigation.navigate('SharedEntries', {
  connectionId: 'uuid-of-connection'
});
```

**Features**:
- List of shared entries (most recent first)
- Entry cards with mood, craving, tags
- Tap to view full entry
- Leave supportive comments
- Empty state with guidance

## Data Flow

### Sharing an Entry

```
User taps share
    ↓
ShareEntryModal opens
    ↓
User selects sponsor
    ↓
useSponsorSharedEntries.shareEntries(connectionId, [entry])
    ↓
Encrypted payload generated (RCSHARE:...)
    ↓
Payload stored locally in sponsor_shared_entries table
    ↓
Success state shown with payload
    ↓
User shares payload via SMS/WhatsApp
    ↓
Sponsor pastes payload into app
    ↓
useSponsorSharedEntries.importPayloads(payloadText)
    ↓
Entry decrypted and stored
    ↓
Entry appears in sponsor's dashboard
```

### Commenting on Entry

```
Sponsor views shared entry
    ↓
Taps "Leave Comment"
    ↓
Writes comment text
    ↓
useSponsorSharedEntries.shareComment(connectionId, entryId, comment)
    ↓
Encrypted comment payload generated
    ↓
Sponsor shares payload with sponsee
    ↓
Sponsee imports comment
    ↓
Comment appears in their journal
```

## Hooks Reference

### useSponsorConnections

```typescript
const {
  mySponsor,        // My sponsor connection (if any)
  mySponsees,       // Array of sponsee connections
  pendingInvites,   // Pending connection requests
  isLoading,        // Loading state
  createInvite,     // (name?) => Promise<{code, payload}>
  confirmInvite,    // (payload) => Promise<void>
  connectAsSponsor, // (payload, name?) => Promise<confirmPayload>
  removeConnection, // (connectionId) => Promise<void>
  loadConnections,  // () => Promise<void>
} = useSponsorConnections(userId);
```

### useSponsorSharedEntries

```typescript
const {
  shareEntries,       // (connectionId, entries[], name?) => Promise<string[]>
  shareComment,       // (connectionId, entryId, comment, name?) => Promise<string>
  importPayloads,     // (payloadText) => Promise<{entries, comments, skipped}>
  loadIncomingEntries,// (connectionId) => Promise<SharedEntryView[]>
  loadCommentsForEntry,// (entryId) => Promise<SharedCommentView[]>
} = useSponsorSharedEntries(userId);
```

## Common Issues & Solutions

### Issue: "No sponsor connected" message

**Cause**: User hasn't completed sponsor connection flow  
**Solution**: Guide user to create invite → share with sponsor → paste confirmation

### Issue: Payload import fails

**Cause**: Wrong shared key or corrupted payload  
**Solution**: Ensure both parties are using same connection (invite code matches)

### Issue: Unread count not updating

**Cause**: Read/unread status not tracked in database  
**Solution**: Add `read_at` column to `sponsor_shared_entries` table (future enhancement)

### Issue: Modal doesn't close on Android

**Cause**: Missing `onRequestClose` handler  
**Solution**: Always add `onRequestClose={onClose}` to Modal components
