# Navigation Integration Guide

## How to Use the New Sponsor Sharing Screens

### Option A: Direct Replacement (Recommended)

Replace the old screens in your navigation files:

#### 1. Find Your Navigator File

Likely located at:
- `apps/mobile/src/navigation/ProfileNavigator.tsx` or
- `apps/mobile/src/navigation/AppNavigator.tsx`

#### 2. Update Imports

```typescript
// ❌ OLD
import { SponsorScreen } from '../features/sponsor/screens/SponsorScreen';
import { SharedEntriesScreen } from '../features/sponsor/screens/SharedEntriesScreen';

// ✅ NEW
import { SponsorScreenModern } from '../features/sponsor/screens/SponsorScreenModern';
import { SharedEntriesScreenModern } from '../features/sponsor/screens/SharedEntriesScreenModern';
```

#### 3. Update Screen Definitions

```typescript
// ❌ OLD
<Stack.Screen 
  name="Sponsor" 
  component={SponsorScreen}
  options={{ title: 'Sponsor' }}
/>

<Stack.Screen 
  name="SharedEntries" 
  component={SharedEntriesScreen}
  options={{ title: 'Shared Entries' }}
/>

// ✅ NEW
<Stack.Screen 
  name="Sponsor" 
  component={SponsorScreenModern}
  options={{ 
    title: 'Sponsor',
    headerShown: false  // Screen has its own header
  }}
/>

<Stack.Screen 
  name="SharedEntries" 
  component={SharedEntriesScreenModern}
  options={{ 
    title: 'Shared Entries',
    headerShown: false  // Screen has its own header
  }}
/>
```

```typescript

  NEW_SPONSOR_UI: __DEV__ || true,  // Enable in dev, rollout in prod
};

// In navigator:
import { FEATURES } from '../config/features';

<Stack.Screen 
  name="Sponsor" 
  component={FEATURES.NEW_SPONSOR_UI ? SponsorScreenModern : SponsorScreen}
  options={{ 
    title: 'Sponsor',
    headerShown: !FEATURES.NEW_SPONSOR_UI
  }}
```

## Common Issues

### Issue: "Cannot read property 'navigate' of undefined"

**Cause**: Navigator context not available  
**Solution**: Wrap app in NavigationContainer

### Issue: Screen flashes white before rendering

**Cause**: Missing background on screen  
**Solution**: Already fixed - screens have LinearGradient backgrounds

### Issue: Android back button doesn't work

**Cause**: Missing hardware back handler  
**Solution**: Add to screen:

```typescript
import { BackHandler } from 'react-native';

useEffect(() => {
  const backHandler = BackHandler.addEventListener('hardwareBackPress', () => {
    navigation.goBack();
    return true;
  });
  return () => backHandler.remove();
}, [navigation]);
```

### Issue: TypeScript errors on navigation.navigate

**Cause**: Type definitions don't match route params  
**Solution**: Update `ProfileStackParamList` with correct params
