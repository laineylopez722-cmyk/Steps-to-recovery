## 📝 Notes for Next Developer

### Important Context:
1. **Hooks are the source of truth**: All data flows through `useSponsorConnections` and `useSponsorSharedEntries`
2. **Payloads are strings**: Encrypted entries are base64-encoded strings prefixed with `RCSHARE:`
3. **Local database**: Uses SQLite via `useDatabase` context
4. **No backend calls**: Everything is device-to-device via share payloads
5. **Glassmorphic theme**: Use `darkAccent` tokens from `design-system/tokens/modern`

### Known Limitations:
- Comment feature generates payload but UI doesn't show text input (intentional - keeps modal simple)
- Unread tracking is client-side only (no sync)
- Connection flow requires manual payload exchange (future: QR codes)

### Testing Tips:
- Use two devices/emulators for real testing
- Copy payloads via Share → Notes → Manual paste for testing
- Check SQLite database with `expo-sqlite` inspector
- Test with long entries (10,000+ chars)
- Test with special characters and emojis
